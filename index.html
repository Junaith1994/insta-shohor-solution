Question-1: How does JavaScript work?

Answer: Javascript is a single threaded, interpreted language which executes the code line by line. It's basically a client-side language because it runs inside the client-side browser. Most of the browsers have their own inbuilt javascript engine to execute the javascript code. Such as google chrome has V8 engine, Mozilla has spider-Monkey. When a browser gets the javascript source code, the browser javascript engine firstly parse the source code line by line, if any error or mistakes found in the code then it throws an error and immediately stops the execution. After successfully parsing the code the parser creates an Abstract Syntax Tree (AST) to convert the code into machine readable bytecodes. After converting the code into machine readable language then the code is finally executed and gives the output.

Question-2: Differences between Local Storage and Session Storage

Answer: 1. Local storage maximum capacity is 10MB while the session storage maximum capacity is 5MB.

2. Local storage data never expires even after closing the tab or window while on the other hand session storage data expires as you close the tab or close the window.

3. Local storage data is accessible from any window or tab but the session storage data is accessible from the same tab only.

4. You can store data like user preferences or any data you want to retrieve in local storage, on the other hand session storage is useful for storing any sensitive information like users login credentials etc.

5. Local storage is useful for storing user related data while the Session storage is useful for improving your web application performance.
